## 2. Qualité du code PowerShell

### 2.1 Points forts

- **Architecture modulaire** : `lib.ps1` centralise 22 fonctions réutilisables (logs, connexions, mapping OS, résolution VLAN)
- **Worker pool pattern** : Step3 utilise des workers persistants avec file-system queue (pending/processing/done/failed) — robuste pour la reprise
- **Compatibilité PowerShell 7** : Stratégie d'import en 3 niveaux (Standard → WindowsPowerShell → SkipEditionCheck) avec détection des modules Windows-only (VirtualMachineManager, Veeam.Backup.PowerShell, FailoverClusters)
- **Résolution VLAN multi-strate** : DVS VlanId → VlanConfiguration regex → Standard PortGroup → ExtensionData backing → nom du port group → fallback
- **Sécurité** : `ConvertTo-HtmlEncoded` pour les templates email, pas de mots de passe en dur dans `config.psd1`, credentials sollicités au runtime
- **Gestion d'erreurs** : try/catch cohérents, logging structuré avec `Write-MigrationLog`

## 5. Problèmes identifiés

### 5.1 Bloquant
- **`Read-Host` ligne 117 de `run-migration.ps1`** : bloque toute exécution automatisée/non interactive entre step2 et step3. À remplacer par un paramètre `-SkipManualValidation` ou `-NonInteractive`.

### 5.2 Important
- **Absence de comment-based help** : seul `Test-HyperVNodeReadiness.Tests.ps1` a un bloc `<# .SYNOPSIS #>` complet. Tous les scripts de production devraient en avoir un.
- **Tests manquants sur les scripts critiques** : `step3-MigrateVM.ps1` (1532 lignes, zéro test), `run-migration.ps1` (688 lignes, zéro test unitaire)
- **Paramètres inutilisés** : 5 fonctions avec des paramètres définis mais non utilisés (signalés par PSReviewUnusedParameter)

### 5.3 Recommandations
- Ajouter `PSScriptAnalyzerSettings.psd1` pour supprimer les faux positifs et formaliser les règles acceptées
- Ajouter code coverage dans CI (Pester `-CodeCoverage`)
- Uniformiser `exit 1` vs `throw` : `exit 1` est utilisé dans `run-migration.ps1` alors que `throw` est plus idiomatique
- Ajouter `.gitignore` pour `*.log`, `node_modules/`, `__pycache__/` (actuellement minimaliste)
- Refactorer `step-XX-StartVM.ps1` pour réduire les 12 occurrences de `PSUseUsingScopeModifierInNewRunspaces`
- Ajouter des tests de charge (multi-VM, worker pool)
